```

結局47都道府県の教育委員会が出してる入試の実施要項を全部自分で読んだ。県によって「実技教科が2倍」
```

### ⑧AIに聞くと内申点も一発で返ってくる
```
MCPサーバーには「評定から内申点を計算する」だけじゃなく、「目標点から逆算する」「複数県を比較する」
「週次の学習計画を組む」というツールも用意してる。だから「内申が180点必要なんだけど今どのくらい頑張れば
いい？」みたいな会話をAIとするだけで、47都道府県ぶんの正確な数字ベースで答えが返ってくる。
```
ネタ源: `/api/mcp`の実装ツール一覧（calculate_naishin/reverse_calc/compare_prefectures/build_study_plan・
`/developers`ページに記載の既存実装）。

### ⑨偏差値の「上位%」と内申点の「上位%」は仕組みが違う
```
偏差値の「上位16%」みたいな数字は、正規分布という統計モデルの計算結果（平均50・標準偏差10という前提での
理論値）。一方で内申点の「全国で上位何%」は、実際に計算した人たちのデータを集めないと出せない
（理論では出ない）。だからサイトでは後者は同意した人の匿名データが一定数貯まってから初めて表示するようにしてる。
```
ネタ源: `src/lib/hensachi.ts`（正規分布ベースの理論値）と`src/lib/stats-aggregation.ts`
（`computePercentileRank`=実データベースの実測値・k-匿名性で30件未満は非表示）の設計の違い。

## バッチ5（2026-07-15・3本／通算15/15・上限到達）

### ⑬最初から47都道府県対応にした理由
```
自分の県だけ内申点計算サイトを作ってもよかったけど、それだと市場が47分の1になる。しかも都道府県ごとに
計算方式が違うという事実自体が、他の人が簡単に真似できない参入障壁になる。だから最初から全国対応で
作ることに決めた。1県分より47県分のほうが、実は大変さの割に価値が大きいと思ってる。
```
ネタ源: サイト全体の設計方針（47都道府県対応がコンテンツ構成の一次前提・既存実装で確認可能）。

### ⑭ブログ記事のアクセスがAIに吸われてた話
```
ブログ記事のタイトルを2回改善したのに、クリック率が1.16%→1.58%で頭打ちになった。調べたら、
検索結果の上でAIの要約（AI Overview）がうちの記事の内容をそのまま答えてしまっていて、
クリックされる前に読者の疑問が解決してた。記事自体は悪くないけど、検索結果の仕組みが変わった。
今はブログの改修より、AIが要約しにくい「計算ツール」のほうに力を入れてる。
```
ネタ源: [[session-2026-07-05-fullaccel-megaplan]]記載の実測（all-3記事タイトル2回刷新でCTR
1.16%→1.58%止まり・AI Overview吸収を実測で確認した根拠に基づく戦略転換）。

### ⑮誰のために作ってるか、実は途中で変わった
```
最初は受験生本人が使うことを想定して作ってた。でも実際は、月謝や塾を決めるのは保護者。
だから最近は「子どもの計算結果を保護者に共有する」導線を強化してる。作ってるものは同じ内申点計算でも、
「誰の意思決定を助けるか」で機能の優先順位が全然変わってくるのが面白い。
```
ネタ源: サイト全体の保護者向け導線（ParentShareLinkButton等の既存実装・[[oneshot-revenue-architecture]]の
「保護者起点」戦略方針）。
