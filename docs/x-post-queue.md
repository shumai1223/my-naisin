### ③47都道府県で内申点の中身が違う
```

独自の指標を使う。同じ「内申点」でも中身が全然違うので、全国共通の答えは存在しない。
```
ネタ源: src/lib/prefectures.ts実データで検証済み（東京=targetGrades:[3]・practicalMultiplier:2／
埼玉=targetGrades:[1,2,3]／神奈川=独自のS値方式で別エンジン実装）。

## バッチ2（2026-07-15・3本／通算6/15）

### ④匿名統計の「見せない」設計
```
サイトに「全国の内申点・偏差値の統計」を出す機能を作った。ただし、集まった件数が30件未満の項目は
表示しない設計にした（k-匿名性というやつ）。少ない人数の平均を出すと個人の値がバレかねないので、
データが少ないうちは正直に「収集中」と出す。見栄えより先に守るべきものがあると思ってる。
```
ネタ源: `src/lib/stats-aggregation.ts`の`STATS_MIN_SAMPLE_SIZE = 30`（実装済みの既存基準）。

### ⑤AIが直接答えを引ける設計にした
```
このサイト、人間向けのページとは別に、AIエージェント向けの窓口（MCP）を用意してる。ChatGPTやClaudeが
「◯◯県の内申点の計算方法」を聞かれたとき、うちのサーバーに直接問い合わせて正確な数字を返せるように
JSON-RPCで公開してる。47都道府県ぶんの計算式を、AIに"教え込む"んじゃなくてAPIで"引かせる"発想。
```
ネタ源: `/api/mcp`（`src/app/api/mcp/route.ts`・tools/list等の既存実装。V-10で組み込みサンプルも整備済み）。

ネタ源: [[session-2026-07-13-gmail-mcp-b2b-day1-link-wall]]記載の初lead_submit（兵庫・登録→開封→クリック51秒）。

## バッチ3（2026-07-15・3本／通算9/15）

### ⑦47都道府県分、全部一次資料を読んだ
```
内申点の計算方式、ネットのまとめ記事だと情報が古かったり間違ってたりすることがあるので、
結局47都道府県の教育委員会が出してる入試の実施要項を全部自分で読んだ。県によって「実技教科が2倍」
「中1は対象外」「独自の指標を使う」など仕組みがバラバラで、コピペで済むところは1つもなかった。
```
ネタ源: `src/lib/prefectures.ts`（47都道府県ぶんの`sourceUrl`/`sourceTitle`が個別に入っている実装）。

### ⑧AIに聞くと内申点も一発で返ってくる
```
MCPサーバーには「評定から内申点を計算する」だけじゃなく、「目標点から逆算する」「複数県を比較する」
「週次の学習計画を組む」というツールも用意してる。だから「内申が180点必要なんだけど今どのくらい頑張れば
いい？」みたいな会話をAIとするだけで、47都道府県ぶんの正確な数字ベースで答えが返ってくる。
```
ネタ源: `/api/mcp`の実装ツール一覧（calculate_naishin/reverse_calc/compare_prefectures/build_study_plan・
`/developers`ページに記載の既存実装）。

### ⑨偏差値の「上位%」と内申点の「上位%」は仕組みが違う
```
偏差値の「上位16%」みたいな数字は、正規分布という統計モデルの計算結果（平均50・標準偏差10という前提での
理論値）。一方で内申点の「全国で上位何%」は、実際に計算した人たちのデータを集めないと出せない
（理論では出ない）。だからサイトでは後者は同意した人の匿名データが一定数貯まってから初めて表示するようにしてる。
```
ネタ源: `src/lib/hensachi.ts`（正規分布ベースの理論値）と`src/lib/stats-aggregation.ts`
（`computePercentileRank`=実データベースの実測値・k-匿名性で30件未満は非表示）の設計の違い。

### ⑩営業リストも自分で1件ずつ実在確認して作った

フォーム誤検出→訂正した実際の教訓）。

### ⑪一番複雑だった県
```
13都道府県ぶん計算機を作った中で、一番複雑だったのは大阪府。学力検査と内申点の比率が「タイプⅠ（7:3）」
から「タイプⅤ（3:7）」まで5パターンあって、志望校ごとにどのタイプかが違う。単純な一発計算式じゃなく、
選択式のUIにしないと正しく表現できなかった。
```
ネタ源: `/osaka/total-score`ページの実装（タイプⅠ〜Ⅴの5パターン選択式・既存コード確認済み）。

### ⑫サーバーの話（技術寄り）
```
このサイト、Cloudflare Workers上で動いてる。名簿やクリックログはD1（CloudflareのサーバーレスDB）に
貯めてて、Next.jsをOpenNextでWorkers向けにビルドしてる。個人開発でここまで無料〜低コストで
本番運用できるのはありがたい。
```
ネタ源: `wrangler.jsonc`（LEADS_DBバインディング・OpenNext設定の既存実装）。
