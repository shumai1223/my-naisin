### 応募書類ドラフト

```
【作品名】My Naishin（内申点・偏差値計算サイト＋API/MCPサーバー）

【概要】
全国47都道府県の公立高校入試における「内申点」の算出方法は、教育委員会ごとに
バラバラで、生徒・保護者が正確に把握するのは困難です。私はこの課題を解決するため、
47都道府県すべての教育委員会が公表する入学者選抜実施要綱を自分で読み解き、
内申点・偏差値・総合得点の計算ロジックを実装した無料サイトを個人開発しました。
現在は月間数千クリック規模で受験生・保護者にご利用いただいています。

【技術的な工夫点】
・47都道府県の異なる制度を、対象学年・学年別倍率・実技教科倍率・満点という
  共通パラメータで表現できるものは1つのデータ構造に統一し、無理に統一できない
  8都道府県（東京・神奈川・大阪・愛知・千葉・埼玉・福岡・北海道）は個別実装に
  正直に分離するという設計判断で、複雑なドメインをコードに落とし込みました。
・REST API・OpenAPI 3.1仕様・MCP（Model Context Protocol）サーバーとして
  計算ロジックを無料公開し、AIエージェントからも直接呼び出せるようにしています。
・uniqueness.test.ts（量産コンテンツ検知）・internal-link-graph.test.ts（孤児
  ページ検知）・rich-results-*.test.ts（構造化データ網羅監査）など、1人で
  数十〜100ページ規模のサイトを保守するための独自の品質担保テストを整備しています。

【開発のきっかけ】
自分自身が高校受験生として、内申点の計算方法が都道府県ごとに全く違うことに
戸惑った経験がきっかけです。同じ悩みを持つ全国の受験生・保護者の力になりたいと
思い、独学でプログラミングを学びながら開発しました。

【今後の展望】
教育データのオープン化・機械可読化をさらに進め、教育行政研究者や他の開発者にも
活用してもらえるデータ基盤を目指しています。
```
