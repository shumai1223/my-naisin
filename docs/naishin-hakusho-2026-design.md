# 内申点白書2026 設計書（V-9・2026-07-15）

[[fable5-fullaccel-backlog-2026-07]] V-9。既存の[/stats](/stats)（常時更新の匿名統計ダッシュボード）とは別に、
一定サンプルが貯まった時点で**季節発行の分析レポート（白書）**として編纂する構想の設計のみをここに置く。
本ドキュメントは構成・発行条件の**設計**であり、実装（レイアウト・生成バッチ）は別タスク。価格帯は書かない（👤裁定事項）。

## 1. /statsとの違い（なぜ別物として作るか）

| | /stats（既存・常時稼働） | 内申点白書（本設計） |
|---|---|---|
| 更新頻度 | リクエストのたびに最新集計（force-dynamic） | 季節ごと（学期末・入試シーズン後など）にスナップショットを固定 |
| 形式 | Webページ+CSV | PDF/note記事等の「読み物」として編集 |
| 目的 | 透明性・引用可能な一次データの常設公開 | メディア・模試会社への売り込み材料（[[session-2026-07-13-gmail-mcp-b2b-day1-link-wall]]§5の編集リンク獲得戦略・V-4メディアリストの着地資産） |
| データソース | 同一（stats_submissionsテーブル・k-匿名性基準も共通） | 同一 |

白書は/statsの「読み物版・特定時点のスナップショット」であり、集計ロジック・k-匿名性基準は完全に共有する（二重実装しない）。

## 2. k-匿名性基準（既存基準にそのまま準拠）

`src/lib/stats-aggregation.ts`の`STATS_MIN_SAMPLE_SIZE = 30`をそのまま踏襲する。
白書のどのセル（都道府県×指標×学年等のクロス集計）も、**サンプルサイズ30件未満は非公開**（「データ収集中」表示、数値の推測を許さない）。
これは新しい基準を作らず、既存の一次ソース（stats-aggregation.ts）を白書側が参照する設計とし、
将来この定数が変更された場合も白書側の実装が自動的に追従するようにする（値のハードコード禁止）。

## 3. 必要サンプル数の統計的根拠

平均値・パーセンタイルの**セル単位**の下限はk-匿名性基準（n≥30）で決まるが、白書として「全国の内申点分布」を
意味のある精度で語るには、単純な下限突破以上のサンプル数が要る。二項比率（例:「偏差値60以上の割合」）の
95%信頼区間の半幅は概ね `1.96 × √(p(1-p)/n)` で近似できる。p=0.5（最も分散が大きいケース）のとき：

| サンプル数n | 95%信頼区間の半幅（目安） |
|---|---|
| 30（k-匿名性の最低ライン） | 約±17.9pt（白書の「全国傾向」として語るには粗すぎる） |
| 100 | 約±9.8pt |
| 400 | 約±4.9pt |
| 1,000 | 約±3.1pt |

**白書「全国編」の発行ライン**は、全国合計の有効回答（stats_submissions）が**n≥1,000**に達した時点とする
（±3pt程度の精度で「全国の内申点は◯◯点台が最も多い」といった記述が誠実に書けるライン）。
**都道府県別セクションを含める場合**は、対象都道府県ごとにn≥100（±10pt程度）を個別の掲載可否ラインとし、
届かない県は「収集中」として掲載を見送る（全国合計n≥1,000を満たしても、収載する県別データは県ごとに再判定する）。

## 4. 発行トリガー（機械判定できる形で定義）

以下をすべて満たした時点で「着手可能」と判定する（自動発行ではなく、判定結果をworklog/週次KPIで人間に提示するのみ）：

1. `stats_submissions`の指標別有効件数（metric='naishin'）が**全国合計n≥1,000**。
2. 直近30日で新規提出が継続している（収集が止まっていない＝スナップショットの鮮度を担保）。
3. 3指標（naishin/hensachi/total-score）のうち、白書に含める指標は個別にn≥1,000を満たすもののみ
   （全指標が同時に閾値に達するのを待たず、達した指標から順次「◯◯編」として分割発行できる設計にする）。

実装時は`stats-aggregation.ts`に判定用の純粋関数（例: `evaluateHakushoTrigger(counts: Record<StatsMetric, number>)`）を
追加し、`weekly-kpi-report.ts`（[[V-6]]で拡張済み）に1行追加する形が既存パターンと整合する（新規スクリプトを増やさない）。
**現状（2026-07-15時点）はstats_submissions累計1件**であり、発行ラインには遠く及ばない。TIER V完了時点でのタスクは
この設計のみに留め、実装（判定関数・週次KPIへの結線）はn数が閾値に近づいてから着手する（時期尚早な実装を避ける）。

## 5. 構成案：保護者版

対象読者：中学生の保護者。目的：「うちの子の内申点は全国的にどのくらいの位置か」を知ってもらい、LINE名簿・
保護者リード導線への自然な入口にする（[[monetization-reality-2026-06]]の保護者起点C_pと同じ導線思想）。

1. 導入：内申点とは何か・なぜ都道府県で仕組みが違うのか（1ページ・専門用語なし）
2. 全国の内申点分布（n≥1,000達成後の全国編データ・ヒストグラム相当の記述）
3. 都道府県別の傾向（n≥100を満たす県のみ・断定的な「順位」表現は避け「分布の幅」で語る）
4. 「うちの子の内申点」を計算してみる（/への導線・既存のUnlockGate/全国統計パーセンタイル機能への誘導）
5. 内申点を上げる一般論（既存blog/ツール内コンテンツの要約・新規断定コンテンツは作らない）
6. 出典・方法論（次項）

## 6. 構成案：事業者版

対象読者：模試会社・教育メディア・EdTech SaaS（[[V-3]]/[[V-4]]で構築した送付先リストの着地資産）。
目的：一次データの引用元としての権威付け・API/データ連携（[[V-5]]/[[V-10]]）への橋渡し。

1. エグゼクティブサマリー（1ページ・数字を先出し）
2. データセットの概要：収集方法（オプトイン制）・件数・期間・k-匿名性基準の明記（第2章の内容を要約）
3. 全国編：指標別分布（naishin/hensachi/total-score）
4. 都道府県別編：n≥100を満たす県の詳細（今後の増補を前提に「Vol.1で公開できるのはN県」と明記し、
   隔月・四半期などで追補版を出す余地を残す）
5. 方法論の詳細（k-匿名性の考え方・除外基準・満点混在の扱い＝/statsのMETRIC_META注記を踏襲）
6. 引用・出典表記のガイドライン（[[N-8]]の既存「方法論開示」ページと重複させず参照リンクで済ませる）
7. データ連携のご案内（API/MCPへの導線・[[V-5]]の/partnerページ・[[V-10]]の/developersページへリンク。
   価格・料率には触れない＝👤裁定事項）

## 7. 捏造防止ガード（既存原則の継承）

- 学校別ボーダー・実在校の合否断定は一切含めない（[[prefecture-exam-systems-verified]]の既存方針を継承）。
- サンプルサイズ不足のセルは「収集中」と明記し、推測値・過去の非公式データで埋めない。
- ¥金額・収益予測は書かない（[[no-revenue-projections-guideline]]）。事業者版の「データ連携のご案内」も
  価格には触れず、料金は/developersの料金表・👤との個別相談に誘導する。
