## 本文ドラフト

### タイトル案: 47都道府県ぶんのコンテンツを「コピペ量産」にしないための自動テスト

中学生向けの内申点・偏差値計算サイト「My Naishin」は、47都道府県それぞれの解説ページ・FAQを
持っています。しかしこの手のサイトには構造的な罠があります——**都道府県名だけ差し替えた
「find & replaceコンテンツ」になりやすい**、という罠です。

#### なぜ危険か

検索エンジンは、テンプレートに変数を差し込んだだけの量産コンテンツ（scaled content）を
評価しない、あるいはペナルティ対象にすることがあります。47ページも似た構造のページを
作れば、うっかり「県名以外はほぼ同じ文章」になっていないか、目視だけでは見落としが起きます。

#### 解決策: 「文字列としての重複」を機械的に禁止するテスト

そこで、各都道府県の overview（概要文）・FAQ回答・計算手順の説明文を、**すべて配列に集めて
文字列として完全一致するものがないか**を検証するテストを書きました。

```typescript
function expectAllUnique(label: string, values: string[]) {
  const seen = new Map<string, number>();
  const dups: string[] = [];
  for (const v of values) {
    const n = (seen.get(v) ?? 0) + 1;
    seen.set(v, n);
    if (n === 2) dups.push(v.slice(0, 40));
  }
  expect({ label, dups }).toEqual({ label, dups: [] });
}
```

これを「全県のoverviewが重複しない」「全県のFAQ回答が重複しない」「解説の展開(flow)が重複
しない」という複数の観点でそれぞれ実行します。もし誰か（人間でもAIエージェントでも）が
新しい県のページを追加する際にテンプレートをコピーして中身を書き換え忘れると、このテストが
**即座に赤くなって教えてくれます**。

もう一つ、地味だが効果的なチェックが「各県のoverviewに自県名が含まれているか」という
アサーションです。これは「そもそも県名を書き換えるのを忘れていないか」という初歩的だが
実際に起きうるミスを機械的に防ぎます。

#### 工夫: あえて"完全一致"に留めた理由

文章の類似度を測るなら、レーベンシュタイン距離やコサイン類似度のような「あいまい類似度」
判定も考えられます。しかしこのサイトでは**あえて文字列の完全一致だけ**を見るシンプルな
実装にしました。理由は2つあります。

1. 47都道府県は制度が本当に違うので、正しく書けば文章が完全一致することはまず起こらない
   （＝誤検知が少ない）
2. 判定ロジック自体がシンプルなら、テストが落ちた理由を人間がすぐ理解できる
   （あいまい類似度は「なぜ引っかかったか」の説明コストが高い）

「複雑な検出精度」より「誤検知が少なく・原因がすぐ分かる」ことを優先した結果、この
シンプルな実装に落ち着きました。

#### まとめ

47都道府県のような「構造は似ているが中身は違う」大量ページを扱うプロジェクトでは、
テンプレート流用の検出をレビュー任せにせず、CIで機械的に強制するのが効果的だと感じています。
特にAIエージェントに大量ページの追加を任せる運用（本サイトは実際にAIエージェントの自律ループで
コンテンツを量産しています）では、このての「品質のゲート」がないと、気づかないうちに
コピペ量産サイトになってしまうリスクがあります。
