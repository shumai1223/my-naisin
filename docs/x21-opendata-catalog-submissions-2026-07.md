# X-21 オープンデータカタログ・API集約サイト掲載申請 下書き（2026-07-23）

[[fable5-fullaccel-backlog-2026-07]] TIER X - X-21。目的は中立的な第三者カタログからの被リンク獲得。
**掲載申請自体（GitHub PRの送信）は👤権限**（[[fable5-loop-protocol]]の禁止事項＝GitHubアカウント作成・
コミット履歴の公開はPII/実在感の露出を伴うため）。ここでは対象カタログの調査と、実際に貼り付けて
使えるPR用の追加行・PR説明文の下書きまでを行う。

## 調査結果サマリー

- **政府系オープンデータカタログ（data.go.jp・各自治体ポータル等）は対象外と判断**: これらは行政機関・
  自治体自身が保有するデータセットの公開基盤であり、民間・個人運営サイトのデータセット掲載を
  受け付ける仕組みではない（調査の結果、掲載申請窓口自体が存在しないことを確認）。無理にここへ
  申請しようとするのは時間の無駄と判断し、対象から除外する。
- **RapidAPI**: 自己サービス型のAPI公開マーケットプレイスで、掲載自体はアカウント作成→API登録
  （BaseURL・エンドポイント定義・課金プラン設定）というオンボーディングフローが必要。単純な
  「フォーム記入で終わる掲載申請」ではなく、実質的にRapidAPIのゲートウェイ経由でAPIを再ホストする
  設定作業に近い。良い候補だが「低心理障壁のフォーム申請」という今回のスコープよりも重く、
  アカウント作成を伴う👤裁定が必要な候補のため今回は見送り、記録にのみ留める。
- **GitHub上の"awesome"型キュレーションリスト（PRベースの掲載申請）が最も低心理障壁で適合**。
  以下2件を具体的な下書きまで作成した。

## 候補1: awesome-japan-opendata（最有力）

- URL: https://github.com/japan-opendata/awesome-japan-opendata
- 掲載方法: README冒頭に「一般・自治体・官公庁の方からの情報の追加・修正、リンク切れの報告等を
  歓迎しています。Issues または Pull Requests からお送りください」と明記。PRベースで受け付け実績あり。
- セクション構成: 全国／政府・国／地方自治体（都道府県別）／**コミュニティ、企業、大学**／
  プラットフォーム・ソフトウェア／ダッシュボード・アプリケーション、の大分類。「文化・芸術・教育」
  セクションは「全国」配下にあり文部科学省等の政府系エントリが中心。**個人・民間運営の当サイトは
  「コミュニティ、企業、大学」セクションが適合先**と判断。
- 既存エントリの記法例: `[子供の学び応援サイト掲載コンテンツ情報](https://www.mext.go.jp/a_menu/ikusei/gakusyushien/mext_00975.html) (文部科学省)`

## 候補2: awesome-japan（snamiki1212）

- URL: https://github.com/snamiki1212/awesome-japan
