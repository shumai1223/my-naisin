# 商談副操縦士 (meeting-copilot)

オンライン商談(Google Meet/Zoom)中に画面の隅に置く、リアルタイム回答補助ツール。

**設計思想: 自分が主・AIは横目の補助。** 知っている質問は「即答スクリプト」(遅延ゼロ)、想定外の質問だけAIボタン(2〜5秒)。

## 使い方

1. **商談の10分前**に `copilot-start.bat` をダブルクリック
   - サーバー起動+ブラウザが開く。AIのウォームアップに約40秒かかる(ヘッダーが「AI: 準備完了」になるまで待つ)
2. **イヤホンを外してスピーカー出音**にする(相手の声をマイクで拾って文字起こしするため)
3. 「🎙️ 文字起こし開始」を押してマイク許可
4. 商談中は**基本何もしない(自動回答モードON)**:
   - 発言の区切りごとに自動でAIへ流れ、**毎回**回答ヒントが streaming で現れる(最初の文字まで数秒)
   - 1行目は「そうですね」と言って時間を稼ぎながらそのまま読み始められるセリフになっている
   - 来るとわかってる質問(価格/独占/実績/内製/無料枠…) → **即答スクリプト**をクリック(遅延ゼロ)
   - 手動で聞きたいときは入力欄+⚡手動(Ctrl+Enter)
   - 聞き取りミスは左下の手入力欄で追記できる

## 注意

- 音声・文字はどこにも保存しない(ブラウザを閉じたら消える)。claude CLIがローカルに残すセッションログも起動時・終了時に自動削除する。ただし相手の発言をAI処理するので、**冒頭に「メモにAIツールを使っています」と一言断るのが推奨**(今のビジネスでは普通の断り)
- 文字起こしはChromeのWeb Speech API(要ネット接続・Chrome推奨)
- AI部分は claude CLI を stream-json 常駐モードで使用(APIキー不要・MCP非ロード)。モデルは既定 haiku(速度優先)。`COPILOT_MODEL=sonnet` で品質優先に変更可
- 会話履歴が伸びるほど応答は遅くなっていく(毎ターン履歴を読み直すため)ので、8ターンごとに裏で新しいプロセスをウォームアップして無停止で入れ替える(`COPILOT_ROTATE_AFTER`で調整可)。長い商談でも速度が落ちにくい設計
- **⚠️ Claude Codeと同じセッション上限を共有する**。商談の数時間前からClaude Codeの重い作業(loop等)は止めておくこと。上限に当たるとヘッダーに「AI: ⚠️ session limit…」と出る(60秒ごとに自動リトライ)。その場合も即答スクリプトと文字起こしは動き続ける
- ファクト・交渉方針・相手情報は `prompt.md` に集約。**商談ごとにここを書き換える**
